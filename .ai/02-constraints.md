# 02 · 하드 제약 (위반 시 미완료 처리 — 절대 준수)

## 보안·인증 (교육 목적의 *의도된* 설계)
- 사용자 인증·권한 체계 **구현 금지**. 서비스는 익명으로만 동작한다.
- 게시글 수정/삭제 권한은 **등록된 평문 비밀번호와의 단순 일치**로만 확인한다.
  - 비밀번호 **해싱·암호화·솔팅 도입 금지** (RFP III-커뮤니티 나).
  - "보안 강화" 명목으로 bcrypt/argon2 등을 추가하면 **요구사항 위반**이다.

## 비밀정보 관리
- API 키 · DB 경로 · 외부 인증정보는 **`.env`로만** 관리한다.
- 소스코드 · 커밋 · 저장소에 **하드코딩 절대 금지.**
- `.env`는 반드시 `.gitignore`에 등록하고, 제출 저장소에 포함하지 않는다.
- 키/시크릿을 로그·API 응답·에러 메시지에 출력하지 않는다.

## AI 도구·LLM 비용
- 챗봇 LLM은 **OpenAI `gpt-5-mini`** 만 사용한다.
- 제공된 API Key 범위 내에서 운영하며 **예산 초과 불가** → **토큰 최소화가 필수 요구사항**이다.
- 챗봇 답변은 **제공 JSON 데이터에만 근거**한다. 외부 지식·추측으로 사실을 생성하지 않는다.
