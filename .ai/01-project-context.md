# 01 · 프로젝트 컨텍스트

## 개요
- 서비스: 공공데이터 기반 **서울** 지역 정보 공유 **익명 커뮤니티** + AI 챗봇 (LocalHub)
- 선정 권역: **서울**
- 제공 JSON 데이터: 관광지 · 문화시설 · 축제공연행사 · 여행코스 · 레포츠 · 숙박 · 쇼핑
- 납기: **2026-07-16(목) 15:00** · 착수 후 기능 Freeze

## 기술 스택 (고정 · 임의 변경 금지)
| 레이어 | 스택 | 배포 |
|---|---|---|
| Frontend | Vue.js 3 (SPA, Composition API `<script setup>`, Vue Router) | Netlify |
| Backend | FastAPI + SQLAlchemy ORM | Render |
| DB | SQLite (`posts` 중심) | `.db` 파일 제출 |
| LLM | OpenAI `gpt-5-mini` (챗봇 전용) | — |

## MVP 범위 (MoSCoW)
### Must — RFP 필수, 전부 구현
- 1개 권역(서울) 커뮤니티
- 익명 **카테고리 게시판** CRUD (목록 / 상세 / 작성 / 수정 / 삭제) — 카테고리: 관광지 · 맛집 · 축제행사, 목록은 카테고리·검색어 필터 지원
- 게시글 수정용 **평문 비밀번호**
- 게시글 SQLite 저장
- 챗봇 `POST /api/chat` — 제공 JSON 기반 자연어 질의응답
- 플로팅 챗봇 UI (대화 히스토리 유지, 모바일 대응)
- Netlify / Render 배포 + URL 제출

### Should — 선정 선택기능
- 지도 시각화 (Leaflet) — 관광지 · 맛집 마커
- 게시판 검색
- 게시글 조회수

### Won't — 이번 범위 제외 (스코프 크립 방지)
- 회원가입 / 로그인 · 인증 · 권한 체계
- 선정하지 않은 나머지 선택기능 (실시간 알림, 다국어, 경로 안내, 데이터 시각화 대시보드 등)

> MVP 밖 기능은 제안하지도, 구현하지도 않는다. (착수 후 범위 변경 불수용)

## 데이터 원칙
- 제공 JSON 파일 기반 연동. **직접 공공 API 호출 불필요/금지.**
- 추가 데이터 사용 시 라이선스·공공누리 유형(1~4) 확인 후, 기능명세서에 **출처·라이선스·수집일** 기재.

## 산출물 (누락 시 미완료)
FE Repo · BE Repo(둘 다 `.env` 미포함 확인) · SQLite `.db`(초기데이터 포함) · Netlify URL · Render URL · 기능명세서(데이터 출처·라이선스 목록 포함) · WBS · 발표 PPT
